建立一个系统掌控你的生活

发表于 2019-03-21 | 更新于 2019-10-11 | 分类于 生活|Life | 评论数： | 阅读次数：

我是一个自由职业者，做自由职业有很多时间可以自由支配。但如果你脑中没有一个时钟，很快就会迷失。直到接触[Get Things Done]这本书，作者David Allen创造的GTD任务管理理念。有兴趣的找来读一下也会有不少收获。这本书出版时间是2001年，场景设定是上个世纪末的欧美工作环境，现今智能手机和网络的发展是David Allen当初所预想不到的。之后我查阅很多了任务管理软件的教程，作者依旧局限在将软件是做视作工具来处理工作相关事宜,并不能满足我，所以我产生了搭建个人任务管理系统（Personal Task Management System）的想法，对于这个系统的基本设想：

  • 普适性
    适用于生活各个层面，适用于各类人群
  • 便捷
    伸手可及，移动端电脑端均可操作且同步
  • 高效
    提高工作及生活效率

经过数月对OmniFocus的使用和调校，搭建了一套系统，不仅适用于工作，也包括生活的方方面面。这套系统的效用类似于个人助理，将你纷杂繁复的日常事务中解放出来，专心于眼下的工作，

我们现在的状态

现代社会的发展进步极大的拓宽了我们的生活边界，更好的生活质量的同时，也有有更复杂的挑战：

  • 环境几乎一直在更新变化，目标、产品、合作伙伴、用户、市场、技术都在不断地更新变化，这必然导致各种现有结构、模式、角色和责任的不断调整和适应。
  • 工作界限越来越模糊，各个行业各个部门之间的壁垒正在削弱，带来我们个人工作量的增加。
  • 通讯技术的发展将我们与家人、朋友、同事的距离无限拉近，甚至家中老人都开始上网。我们对周围个体的影响力日益增大，也意味着更多的信息需要去处理。

简而言之，我们的生活和工作正在变得越来越忙，很多人依旧尝试依靠自己的大脑去处理所有的任务，这显然并不可靠。即使你有意识的进行任务管理，但方式依旧停留在上世纪六十年代————基于日历，记录所有的行程安排，根据任务的价值和紧急程度划分为1. 2. 3.优先级。这种方法是对部分任务是有效的，但并不足以解决现今你面对的纷繁复杂的日常事务，而且经常计划赶不上变化。

现在回答三个问题来测试一下：

  1. 你有没有在生活或工作上的约定忘记或记错了时间？
  2. 是否有过“总有什么事萦绕在心头，却又想不起来具体是什么”？
  3. 有没有过“需要用到相机的时候却发现没有电了”或者类似的经历？

如果有以上任意一种，那么你的任务管理仍然有优化的余地。即便你已经照顾到了生活的方方面面，PTM系统依旧会帮到你提高效率。

最好的状态“ 行云流水 心无旁骛 ”

让我们反思一下，如果你在任何情况下、任何时间内都能够随心所欲地控制自己的一切事务，情况会是什么样？如果你的内心世界非常清澈，感受不到任何压力，情况会是怎么样？如果你能够把100%的精力投入到自己选择的工作中而不受任何外界的干扰，情形又会是怎么样呢？

这是有可能实现的。有一种方法可以让你紧紧掌控遍及生活各个角落的所有事情的同时保持放松，消耗最少的气力去完成一切有意义的工作。这种方法能使你在这个错综复杂的世界里，体验到道家所说的“心如止水”的境界，优秀的艺术家则称之为“忘我”境界。也许，你很有可能已经体会过这种美妙的感觉。

在这种工作状态下，你头脑清醒，积极的想法层出不穷。虽然整日事务缠身，却仍然能够头脑清醒、轻松自如地控制和处理一切。同时，无论你在做什么，这种境界都是你的最佳表现方式，是那一时的最好的参与状态。纵然时光流逝，你仍能充分调动自己的注意力。所做的事情也正是你应该做的，而且完全出于你的承诺和兴趣。时刻处于万事俱备 只欠东风的最佳状态。

而要达到这种状态的我们要做的就是解放你的大脑，将它从纷杂的任务中剥离开来，专注到眼前的事情，所以我们就需要建立一个系统来管理所有的事物。

你的大脑是用来进行思考的，它不是你的笔记本。

五步Get things done

在这里我们先引进一个概念：材料(stuff) ————“进入你的精神或现实世界中但尚未找到归属的任何事情/信息，你尚未确定它对你的真正意义、预期结果和下一步具体行动的所有事情。” 材料也就是所有需要你去处理的信息，可以是老板的一个电话，爱人的一句抱怨，新税法的出台，脑中突然闪现的火花。然后将材料通过这5个步骤解决。

  1. 收集：百分百地收集一切有待完成的任务；
  2. 分析：厘清辨明任务的属性；
  3. 整理：将所有任务有体系地组织起来；
  4. 回顾：检查任务的要素；
  5. 执行：选择合适的任务开始行动。

这一套任务管理的逻辑非常符合人性。它并不是凭空构建，而是从实际生活的需求出发，实现我们想达到的效果：活在当下，不被与当前无关的事情所干扰。

收集

最重要的技巧————把一切事物从大脑中赶出去。

大脑中肩负短期记忆的那部分组织——往往负责存储不完整、尚未决定和没有组织安排好的材料——其功能类似于个人计算机中的RAM（随机存储器）。你的意识就好像是计算机的CPU（中央处理器），是一个负责处理的工具，而不是一个存储场所。你每次只能考虑两三件事情。但是那些未处理完的事务仍然存储于短期记忆空间里。而这种存储的能力是有限的也有时效性。要想让大脑高效工作，就像让CPU高速处理任务一样，必须保证RAM里的排队任务尽可能的有序并且精简。很多人终日忙忙碌碌，大脑的存储空间早已不堪重负。他们时常感到心神不宁、坐卧不安、执行能力减弱，这是因为在他们内心深处，这些未完成的是事务无时无刻不在侵扰他们的注意力。即使你在阅读这篇文章的时候，我想你也有走神，脑中浮现一些和本文不相关的事情。

所以收集是最为重要的的一步。我们将材料托管到一个外界的存储器中，释放大脑的空间，更少的压力，更高的效率。

分析

将材料收集起来之后，我们便要尽快或者定期清空它。如果不定期清空收集工具，处理里面盛放的“材料”，那么它只是一间装满杂乱资料的仓库。这里讲的并不是完成掉它们，而是进行分析，理清材料的属性和价值，然后构思好下一步行动。

假设一个很简单的例子：我的倒车雷达坏掉了。首先我要做的是把这个信息收集起来，记到备忘录里， 晚上我开始整理今天的备忘录，找出这条信息，进行分析：

  1. 判断它是否需要行动 》Yes
  2. 判断需要几个步骤 》成立项目
  3. 下一步行动 》上网查找就近4S店信息，致电询问配件库存
  4. 确定时间 》划入日程安排

组织整理

在工作流程图的外围，是8个独立的处理方向，涵盖了处理“材料”后可能遇到的所有情况。将它们综合在一起，就构成了一个完整的系统。这个系统可以组织整理你的当前工作，也可以随时接纳新内容。

其中有个有趣的“2分钟立刻执行”原则，其实不一定特定2分钟，而是说一个短暂的时间不会打断你当下工作的节奏，而又有效的完成了一个有价值的行动。

还有出现的一个新的概念：项目————任何需要多个步骤完成的任务。
我们对项目如此定义的根据是：如果某项工作不能一步完成的话，那么，你就需要建立某种机制来提醒自己还有事情尚在进行中。如果你没有建立这样一个提醒机制的话，这件事情很有可能会回到你的大脑中，重新成为一个盘踞在你大脑中的悬而未决的问题，并时刻干扰你的工作。
事实上，你不可能直接执行一个项目，你所能执行的是与项目相关的下一步行动。当你完成了足够多的正确步骤以后，才能达成项目的预期目标。

回顾

回顾便是定期的检查我们建立的系统，它保证我们所采取的每一步行动都能将事务朝着正确的方向推进。其中包括：

  • 已收集的材料
  • 待办事宜
  • 项目清单
  • 日程表

一切需要采取行动的事情，我们都应当对其进行定期的回顾，以使我们的大脑不再花费精力来记忆和回想这些事情。我们无时无刻不在凭借我们的直觉对某些行动做出快速的判断，要保证这些判断的正确性，我们也必须有机会从更高的角度俯看全局。根据我的经验（这也是成千上万人总结出来的经验），使这些事项转化为行动的关键就是：回顾。

执行

什么是一个好的任务管理系统

在我看来，一个优秀的任务管理系统一定要省心并且可靠。

PTM系统工作流程如图所示，对于我们来说要做的其实只有几个行为习惯：

  • 收集所有”材料”
  • 将材料按照GTD理念进行分解，转为切实的行动步骤。
  • 定期的Review检查，执行下一个动作。

当你做好前两项后就可以专心做事，无需考虑其他。完成当下动作之后，只需要回顾系统按图索骥，抽取下一个动作。化繁为简，不仅提高了工作效率，也降低了难度。

OmniFocus

到现在为止，我还没有讲到具体工具的使用。因为工具是次要的，了解了本质的思想，使用什么工具都没有问题。纵观各种任务管理的工具：OmniFocus、Todoist、Things、org-mode，它们支持的功能也各不相同，Project、无限层级、Defer、Due、Repeat、Context、Label、Tag、Filter、Perspective 等等。
抛开这些名词，我们来分析一下任务系统到底是由什么组成的。在我看来，一个任务软件最本质的东西只有三个：任务、附加元素和过滤器。

任务：一个 最小可执行 的单元即为一个任务。这个是最好理解的，所以不做过多解释。

附加元素：作用在任务上的不同维度的限制。这句话读起来有点拗口，举个简单例子，「去 XXX 取快递（截止今天 10:00）」，其中，「截止今天 10:00」就是在时间维度上对这个任务的一个限制。同样的，「项目」、「标签」等等这些都是作用在某个任务上的附加元素。

过滤器：组合不同附加元素的过滤规则。只有附加元素并没有什么实际的作用，但是有了过滤器之后，就可以编写不同的过滤规则来过滤指定规则的任务。例如，我想知道「最近 10 天内重要的事情」，那么过滤规则可能就是「截止日期 < 10 并且 已标注」。

把上面任务系统的基本三要素运用到 OmniFocus 上，任务就是一个个的单独 Todo item，附加元素就是 Todo item 所属的项目、截止日期、延后日期、标签、标注、重复规则等，过滤器就是透视（Perspective）。

一个任务在时间维度上应该有有截止日期（Due）和安排日期（Schedule）两个元素，Due 表示在某一天之前必须被完成，Schedule 表示被安排到某一天做，所以可以设定如下标签来完成 Schedule 的功能：

Schedule：

  • Today：被安排到今天
  • Recently：最近需要做的事
  • ThisWeek：被安排到本周
  • ThisMonth：被安排到本月
  • ThisSeason: 被安排到本季度
  • ThisYear：被安排到今年
    这样在做计划的时候，打上对应的标签，就可以使用过滤器过滤出对应的任务了，比如我想看「这个月和工作相关的任务」，那么就可以指定 Project 为「Work」项目，标签为「ThisMonth」，这样对应的任务就可以使用这个过滤器过滤出来，具体的过滤器设定下面会讲到。若一项任务没有被打上 Schedule 标签，则该任务不该被认为为一项可执行的任务，因为一项未被安排的任务在未来很有可能会被你忽略，所以，对于要添加进 OmniFocus 的任务，一定要打上 Schedule 标签，否则就不要添加进去。

另外一个可以借鉴的设定就是精力值标签：

Energy:

  • LowEnergy
  • MediumEnergy
  • HighFocus
    例如，我们工作了一整天之后，还剩一段时间才下班，精力已经下降到一个很低的值，这个时候已经不适合完成一些需要非常专注的工作了，但是一些工作上的杂事，例如打印东西，提交报表等简单的工作可以利用这个时间来完成，这个时候我们就可以设定一个过滤器，过滤出 Project 为「Work」，标签为「LowEnergy」的任务。

我们可以看到，标签系统可以极大地扩展附加元素的内容，给过滤器添加更多的过滤维度。标签系统的设定应该根据自己的需求来，如果设定的标签却没有在过滤器中用到的话，那么也仅仅是个任务添加了一个没有用上的附加元素而已，意义不大。

个人任务管理系统流程

在整个 系统流程 中，主要分为四个部分：收集、计划、执行、回顾。

当按照上述思路完成以 OmniFocus 对任务管理系统的搭建后：

每天我必须要完成的具体任务全部都在 Today 过滤器中
只需要根据Routine的提示，便可更新日计划、周计划、月计划
当无聊或想利用碎片时间完成一些简单的事情时，只需要去 EasyDo过滤器中去领取就可以了
阅读清单、购物清单等等只需要去To-Sth过滤器中找就可以了
并且可以从Future过滤器中知道日周月计划

收集

收集一定要做到彻底，百分之百收集未竟之事，无论大小、轻重、缓急，只要是那些等待处理的、需要改进的、引起注意的的事务和信息都要收集起来。

  • 即时收集
    对于突发事件、更新信息、灵感等等，出现当下即进行收集，储存到OmniFocus 的Inbox中。但除非必要，不进行深入处理，避免打断当下工作状态。
  • 整理收集
    整理收集为定期回顾收集，用整块时间专门进行思考检索，完善系统，查缺补漏，

收集之后的分析也是必要的，我们将利用回顾（Review）任务来统一把Inbox中的材料进行整理分析，更新到系统。

计划

很多人都习惯一切跟着感觉走。但是，如果事前已做了计划，你对自己的决定就会表现出十足的信心。你对自己行动的态度将从期望升级为信任，由此将大大提高你行动的速度和效力。

一般来说，如果不知道怎么做计划的话，有一个长期计划和一个短期计划就可以了。长期计划主要是一个大体的方向，可以是年度计划或者月度计划；短期计划则需要明确具体要做哪些事情，可以是周计划或者日计划。

实践过程中使用最多的就是周计划和日计划，以及月计划和年计划。在 OmniFocus 中，我设定了几个项目，分别是 Morning Review、Evening Review、Weekly Review、Monthly Review、Annual Review。

  • Evening Review
    由于我晚上睡得比较晚，所以我会在每天晚上通过 Future过滤器，从ThisWeek 和Recently 标签中的任务挑出作为明天的任务，并将它们打上Today标签。

  • Morning Review
    通过Today过滤器，知悉今天要完成的任务

  • Weekly Review
    通过 Future 过滤器，从ThisMonth 标签中挑选出本周的任务，并将它们打上ThisWeek 标签

  • Monthly Review
    通过 Future过滤器，从ThisYear 标签中挑选出本月的任务，并将它们打上ThisMonth标签

执行

执行应该是整个 系统流程 中最后的一个环节，如果只是计划了，但是最后任务却没有被完成，那做计划也只是白费力气。

执行的关键就是要在对的时间内做对的事，这也是为什么要设置那么多过滤器的原因，在工作的时候就只展示工作相关要做的事，在没有精力的时候就只展示简单易做的事情。

如果你做事情的时候很容易分心，可以参考一下番茄工作法，一般我会设置为专注 40 分钟，然后休息 10 分钟。默认的 25 分钟时间过短，可能刚进入状态就要被打断。

还可以尝试使用「结构化拖延法」（Structured Procrastination）。结构化拖延法就是忽略优先级高的事，而是从小的事，优先级比较低的事情开始做，这样慢慢地进入工作的状态，然后再去完成优先级高的事情。

此外还有很多其他工作方法可以参考，自行选择。

回顾

OmniFocus 3 提供了自带的 检查 透视，建立每个项目的时候可以选择多少天回顾一次。实践上，一般工作上需要每天跟进的项目，会把 检查 的时间设置为每天一次，一般的项目设置为一周 检查 一次即可。为了防止忘记回顾，可以把这个 检查 的任务添加到 Morning Review 或者 Weekly Review 中。
